About

Gisat s.r.o. provides its clients with a wide range of state-of-the-art, high quality, value added, and complete geoinformation services based on the Earth Observation technology

Gisat is the developer and the owner of the Snow Portal.

Snow Portal has been developed to support the European Environment Agency (EEA) in their decisions related to the future European Snow & Ice Cover Monitoring Service being evolved within the Copernicus programme. It shall also support many other international users and serve other purposes worldwide."

Data

For the demonstration purposes of this project data is available for the following area of interest (AOI), sensors/ satellites, and time period:

  • Area of interest: Europe
  • Satellites (sensors): Sentinel-2 (MSI), Sentinel-3 (SLSTR), Aqua & Terra (MODIS)
  • Time period: November 2015 – April 2016, November 2016 – April 2017

Classification

For the purpose of this portal individual scenes has been obtained, processed and classified into several classes depending on the satellite/ sensor:

Aqua, Terra (MODIS) Sentinel-3 (SLSTR) Sentinel-2 (MSI)
Snow (pixels above land covered by snow) Snow (pixels covered by snow) NoCloud (produced from the cloud mask, all pixels outside of the cloud mask, i.e. cloud-free pixels)
NoSnow (pixels above land covered by neither Snow, nor Cloud) NoSnow (pixels covered by neither Snow, nor Cloud) NoCloud (produced from the cloud mask, all pixels outside of the cloud mask, i.e. cloud-free pixels)
Ocean (all pixels above the ocean)
Cloud (pixels above land covered by cloud) Cloud (pixels covered by cloud) Cloud (= cloud mask)
Night (pixels in areas covered by polar night/ night) Night (pixels in areas covered by polar night/ night)
NoData (none of the previous) NoData (none of the previous)

Analyses

There are two types of analysis that can be performed.

Snow & Cloud analysis

(using data from MODIS (Aqua, Terra) and SLSTR (Sentinel-3), and their combinations)

1-day composites are created by merging information from all available single scenes and then cropping the data by using the European land mask. N-day scenes are created by merging the information from 1-day composites.

A single value will be assigned to each pixel based on the information per given pixel collected from all available scenes. The priority of assigning a class value is as follows (left = highest priority, right = lowest priority):

Snow > NoSnow > Ocean > Cloud > Night > NoData

i.e.: If value ‘Snow’ was recorded at least once for a given pixel then value ‘Snow’ will be assigned to that pixel in the resulting composite. Value ‘Night’ can only be assigned to a pixel in a composite if none of the values on the left of ‘Night’ was recorded (i.e. Snow, NoSnow, Ocean, Cloud).

Cloud analysis

(using data from MSI (Sentinel-2))

1-day composites are created by merging the data from all available single scenes. N-day composites are created by merging the information from all available 1-day composites.

Each pixel is assigned a value based on the priority: NoCloud > Cloud. If for a given pixel value ‘NoCloud’ was recorded at least once then this pixel will be assigned value ‘NoCloud’ in the resulting composite.
